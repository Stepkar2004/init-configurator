# project-base — init-configurator (this repo's own copy)

This file is init-configurator's OWN skill, evolved for building the tool. It is NOT
what downstream repos receive: the canonical downstream template is generated by
`beacons.project_skill()` in `src/init_configurator/beacons.py` — edit THAT to change
what a bootstrapped project gets. Every copy evolves with its repo and never syncs back;
divergence is the design, not drift.

## The line (why this repo is shaped the way it is)

**Skills know HOW. Tools know WHETHER. `project.yaml` records WHAT.**
Generation lives in `.claude/skills/` — markdown an agent reads and evolves.
Verification lives in `src/` — deterministic, runs with no AI present. Full rationale:
`docs/design/agentic-base.md`. Never move knowledge from markdown into Python constants;
that direction is the original sin this repo was amputated to undo (2026-07-10).

## Gates (all green before ANY commit)

```
uv run initc run test        # pytest
uv run initc run lint        # ruff check
uv run ruff format --check src tests
uv run initc run typecheck   # mypy strict
uv run initc lint-paths      # no absolute paths
```

`pre-commit install` once; this repo runs its own path-lint hook. If the pydantic model
changed, `uv run initc schema` and commit the regenerated file — CI diffs it on both OSes.

## Chunk discipline (resumability)

A chunk of work is DONE when: gates pass → `docs/state/now.md` and `docs/state/log.md`
updated → committed. Cold start reads `docs/vision.md` → `docs/state/now.md` →
`docs/state/roadmap.md`, in that order. Never end a session with the state files lying.

## The env rule

The same turn any code first reads a new env var: declare it in `project.yaml`, re-run
`initc env`. Doctor's `env-sync` fails on drift, both directions. Vars marked
`secret: true` never get a value in `.env.example` and doctor never echoes them.
`.env` is parsed, never sourced.

## Conventions (from TheBrain2 `knowledge/projects/job-first-pivot.md`)

1. **Stepan owns every line.** AI drafts; nothing merges to main until he can explain the
   change. If he approves fast without understanding, slow him down — that's the
   argmech/VC2 debt lesson. Interviewers probe; the code must be HIS.
2. **Root-relative paths only.** No machine-absolute paths anywhere; this repo builds the
   linter that enforces it. Use `project_root()` / `path_to()` in code.
3. **No global installs.** Deps live in `./.venv`. This repo itself is never dockerized.
4. **Ship visibly:** a milestone isn't done until README shows it (GIF/screenshot) and a
   LinkedIn reflection draft exists. Demo before polish.
5. **A skill edit is a code change.** It ships as a human-reviewed diff — the `evolve`
   skill owns the procedure. New learned claims are marked `raw (YYYY-MM)`; only Stepan
   promotes them to validated.
6. On session start: read the brain note pointer in CLAUDE.md for current project state.

## Module map (src/init_configurator/, post-amputation 2026-07)

| Module | Answers |
|---|---|
| `manifest.py` | is `project.yaml` well-formed? (pydantic models, teaching errors) |
| `doctor.py` | can this machine work here? (ok/warn/fail, every problem prints its fix) |
| `runtimes.py` | per-language CHECKS only: runtime binary, install dir, install_ok |
| `describe.py` | what does this existing repo contain? (drafts a manifest, FILL_ME gaps) |
| `env_contract.py` | `.env.example` templated from the declared contract |
| `runner.py` | run a declared task from anywhere in the tree |
| `path_lint.py` / `paths.py` | forbid absolute paths / make relative ones effortless |
| `beacons.py` | template source for downstream CLAUDE.md/AGENTS.md + project skill |
| `textfile.py` | every generated file is written LF, on every OS |
| `cli.py` | `validate · doctor · env · run · lint-paths · schema · describe` |

There is no `init`, no presets/, no docker generation — deleted 2026-07-10 on purpose;
installing is a declared task (`initc run install`). **Adding a language** = one entry in
`runtimes.RUNTIMES` + one member of `manifest.Language` + one reference file under
`bootstrap/references/`. A test asserts registry and manifest agree.

## Teach-me protocol (fires when Stepan says "I don't understand X" / "teach me X")

Explain PROPERLY, not fast: (1) the concept from first principles, no jargon assumed;
(2) a minimal generic example; (3) the same idea as it appears in THIS repo's code;
(4) common pitfalls/misconceptions; (5) check understanding with 1-2 questions back.
For deep topics, suggest a fresh side session dedicated to teaching. Teaching writes
NOTHING to the repo unless he asks.
